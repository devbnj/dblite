The Problem
  • Need for low cost and easy to understand data transportation service
  • Reduce costs and manage the demand of the underserved segment of data transportation and distribution

The Solution

  • Allows the user to create and maintain multiple networks with multiple channels (SmartPhones, Internet, IoT devices) in a highly safe environment. The small footprint in DataKNOX helps in reducing data and security volatility.
  • Even though your ledger can grow immensely over time, cold storage mechanism in DataKNOX enables the network to run efficiently by pushing older data into vaults, keeping the nodes and network as efficient as it was new.
  • Users can conveniently add journals into the ledger ‘lake’. Multiple copies will minimize data corruption or accidental deletion risks and run adhoc queries at any point in the lifecycle of the ledger.
  • Product users can conveniently audit, track, manage and reconcile ledger entries using ‘analytics’

dltlite is an open source distributed datafile system with high encryption.

Unlike blockchain, dltlite does not have a data structure in blocks. A distributed ledger in our case, is a database spread across multiple sites, regions, and / or participants. This distributed ledger model used in dltlite sounds exactly how you envision a blockchain solution. The technology in this section derives from our well thought out requirements and motivation. dltlite was built extending a few known open source products:

  • Nginx (TLS) for Reverse Proxy
  • Node.JS with Node-RED (TLS), a few encryption/decryption and hashing functions
  • Open source dashboard and charting functions
  • SQLite3 append only embedded database
  • REDIS (fully encrypted), for Pub-Sub (with encryption)
  • API - RESTful (JSON only)
  • Data - Encrypted at rest, in motion and in transient situations
